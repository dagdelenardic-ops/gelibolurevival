Çanakkale 1915: Interactive Battle Map

Çanakkale 1915: İnteraktif Savaş Haritası

A single-page web application that brings the 1915 Gallipoli Campaign to life on an animated, stylized map. Users watch battle phases auto-play, pause at any moment, and inspect individual units, commanders, and tactical context.

1915 Çanakkale Savaşı'nı animasyonlu, stilize bir haritada canlandıran tek sayfalık web uygulaması. Kullanıcılar savaş fazlarını otomatik izler, istedikleri anda duraklatır, birlikleri, komutanları ve taktik bağlamı inceler.

OVERVIEWPROJECT SCOPE

What Is This Project?

Bu Proje Nedir?

An educational, interactive web application that visualizes the entire 1915 Çanakkale (Gallipoli) Campaign on a stylized map. The app auto-plays animated battle phases with unit movements, attack arrows, and naval operations. At any moment, the user can pause and enter Inspect Mode to click on any unit and read detailed information about it.

1915 Çanakkale (Gelibolu) Savaşı'nın tamamını stilize bir harita üzerinde görselleştiren eğitici, interaktif bir web uygulaması. Uygulama; birlik hareketleri, saldırı okları ve deniz operasyonları ile animasyonlu savaş fazlarını otomatik oynatır. Kullanıcı istediği anda duraklatıp İnceleme Modu'na geçerek herhangi bir birliğe tıklayıp detaylı bilgi okuyabilir.

Analogy: Think of it as a "Netflix documentary you can pause and explore." The timeline plays like a film, but pausing turns it into a museum exhibit where you can walk around and read every label.

Benzetme: Bunu "duraklatıp keşfedebildiğin bir Netflix belgeseli" gibi düşün. Zaman çizelgesi film gibi oynar, ama duraklattığında etrafında gezinip her etiketi okuyabileceğin bir müze sergisine dönüşür.

REQ-4MUST

Pause & Inspect Mode

Duraklatma & İnceleme Modu

The most critical interactive feature. When paused, the map becomes a fully explorable tactical board.

En kritik interaktif özellik. Duraklatıldığında harita, tamamen keşfedilebilir bir taktik tahtasına dönüşür.

Pause Behavior

Duraklatma Davranışı

  • Clicking Pause button OR pressing Space freezes all animation instantly
  • Duraklat düğmesine tıklama VEYA Space tuşuna basma tüm animasyonu anında dondurur
  • All attack/movement arrows remain visible in frozen state
  • Tüm saldırı/hareket okları dondurulmuş durumda görünür kalır
  • Unit positions locked at exact animation frame
  • Birlik pozisyonları tam animasyon karesinde kilitlenir
  • Visual indicator that app is paused (pulsing border, "PAUSED" badge)
  • Uygulamanın duraklatıldığını gösteren görsel işaret (nabız sınırı, "DURAKLATILDI" rozeti)

Inspect Panel (on unit click)

İnceleme Paneli (birlik tıklandığında)

  • Unit name and faction
  • Birlik adı ve tarafı
  • Commander name and rank
  • Komutan adı ve rütbesi
  • Strength – approximate troop count
  • Mevcudiyet – yaklaşık asker sayısı
  • Current objective – what this unit is trying to do
  • Mevcut hedef – bu birliğin ne yapmaya çalıştığı
  • Historical outcome – what actually happened
  • Tarihî sonuç – gerçekte ne olduğu
  • Context paragraph – short narrative about this unit's role
  • Bağlam paragrafı – bu birliğin rolü hakkında kısa anlatım

Wrong: Pausing only stops auto-advance to next phase but doesn't freeze unit animation mid-movement.

Yanlış: Duraklatma yalnızca bir sonraki faza geçişi durdurup birlik animasyonunu hareket ortasında dondurmaması.

Right: Pausing freezes the EXACT frame — units mid-march, arrows mid-draw — creating a snapshot the user can explore.

Doğru: Duraklatma TAM kareyi dondurur — yürüyüş ortasındaki birlikler, çizilmekte olan oklar — kullanıcının keşfedebileceği bir anlık görüntü.

Analogy: Like pausing a strategy video game (Total War). The battle freezes, you hover over any regiment, and a panel shows all its stats and context.

Benzetme: Bir strateji oyununu (Total War) duraklatmak gibi. Savaş donar, herhangi bir alaya gidersin ve bir panel tüm istatistik ve bağlamını gösterir.

REQ-6MUST

Structured Data Model (JSON)

Yapılandırılmış Veri Modeli (JSON)

All battle data must be stored in a structured JSON object. This allows the data to be extended, corrected, or translated without touching the rendering code.

Tüm savaş verileri yapılandırılmış bir JSON nesnesinde saklanmalıdır. Bu, verilerin render koduna dokunmadan genişletilmesine, düzeltilmesine veya çevrilmesine olanak tanır.

{
  "phases": [
    {
      "id": "naval-assault",
      "date": "1915-03-18",
      "title": { "en": "Naval Assault", "tr": "Deniz Saldırısı" },
      "narration": {
        "en": "The Allied fleet of 18 warships enters...",
        "tr": "18 savaş gemisinden oluşan Müttefik donanması..."
      },
      "units": [
        {
          "id": "57-alay",
          "position": { "x": 145, "y": 82 },
          "status": "defending",
          "strength": 3000,
          "objective": { "en": "Defend hilltop", "tr": "Tepeyi savun" },
          "outcome": { "en": "Successfully held...", "tr": "Başarıyla tuttu..." }
        }
      ],
      "arrows": [
        {
          "from": { "x": 300, "y": 50 },
          "to": { "x": 220, "y": 140 },
          "type": "naval-advance",
          "faction": "british"
        }
      ]
    }
  ]
}

Analogy: The JSON is the screenplay. The rendering engine is the film crew. A bad screenplay can't be saved by a good camera — get the data structure right first.

Benzetme: JSON senaryodur. Render motoru film ekibidir. Kötü senaryoyu iyi kamera kurtaramaz — önce veri yapısını doğru kur.

Key principle: Every text field that users see (titles, narrations, unit descriptions) must have { "en": "...", "tr": "..." } structure for bilingual support.

Temel ilke: Kullanıcıların gördüğü her metin alanı (başlıklar, anlatımlar, birlik açıklamaları) iki dil desteği için { "en": "...", "tr": "..." } yapısına sahip olmalıdır.

REQ-8SHOULD

Animation & Transitions

Animasyon & Geçişler

  • CSS transitions or GSAP for smooth unit movement
  • Akıcı birlik hareketi için CSS geçişleri veya GSAP
  • Ship paths animated as moving along SVG path curves
  • Gemi rotaları SVG yol eğrileri boyunca hareketle animasyonlu
  • Attack arrows draw progressively (stroke-dashoffset technique)
  • Saldırı okları aşamalı çizilir (stroke-dashoffset tekniği)
  • Explosion/impact indicators at landing beaches
  • Çıkarma plajlarında patlama/etki göstergeleri
  • Sunk ships: fade + tilt animation with ✕ marker
  • Batan gemiler: solma + eğilme animasyonu ile ✕ işareti

Performance note: Prefer CSS animations over JS where possible. Use will-change: transform on animated elements. Keep animated elements under 50 for smooth 60fps.

Performans notu: Mümkünse JS yerine CSS animasyonlarını tercih et. Animasyonlu öğelerde will-change: transform kullan. Akıcı 60fps için animasyonlu öğeleri 50'nin altında tut.

TECH STACKDECISION

Recommended Technology Choices

Önerilen Teknoloji Seçimleri

Option A: Single-File HTML (Simpler)

Seçenek A: Tek Dosya HTML (Basit)

- Vanilla JS + SVG
- CSS animations
- All data in inline JSON
- No build step
- Best for: quick prototype, sharing as single file

Pro: Zero setup, portable, easy to iterate with AI coding tools.

Artı: Sıfır kurulum, taşınabilir, AI araçlarıyla kolay iterasyon.

Option B: React Application (Scalable)

Seçenek B: React Uygulaması (Ölçeklenebilir)

- React + Hooks + JSX
- Framer Motion / GSAP
- Separate JSON data file
- Component architecture
- Best for: extending later, adding features

Pro: Clean state management, reusable components, easier to scale.

Artı: Temiz durum yönetimi, yeniden kullanılabilir bileşenler, ölçeklendirilmesi kolay.

Recommendation: Start with Option A (single-file) to get a working prototype, then migrate to Option B if the project needs to grow.

Öneri: Çalışan bir prototip için Seçenek A (tek dosya) ile başla, proje büyümesi gerekirse Seçenek B'ye geç.
